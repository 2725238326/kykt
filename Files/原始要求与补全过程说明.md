## 二、原始代码中明确要求补全的部分

### 1. `virtual_camera_test.py` 中的内参矩阵构造

原始代码在 `get_camera_intrinsics()` 函数中写到：

```python
K = []
raise NotImplementedError("请构建内存矩阵")
```

这里的真实意图应为“请构建内参矩阵”，即要求补出相机内参矩阵 `K`。

应补全的核心内容为：

```python
K = np.array([
    [f_pixels, 0, cx],
    [0, f_pixels, cy],
    [0, 0, 1]
], dtype=float)
```

### 2. `virtual_camera_test.py` 中的透视除法

原始代码在 `project_points()` 函数中写到：

```python
u = v = 0
raise NotImplementedError("请完成透视除法")
```

这里明确要求完成三维点投影到二维像素坐标中的最后一步，即透视除法：

```python
u = points_img_homo[0, :] / points_img_homo[2, :]
v = points_img_homo[1, :] / points_img_homo[2, :]
```

这两处是原始代码中最明显、最核心的“留空补全”部分。

## 三、原始代码中虽然未显式留空，但实际需要修正的部分

### 1. `feature_matching.py` 中的可视化参数不一致

原始代码中调用方式为：

```python
visualize_matches(..., N=10)
```

但函数定义中并没有 `N` 这个参数，因此原脚本会直接报错。

这部分虽然不是“留空题”，但属于必须修正的运行性错误。

### 2. 图片路径与运行入口不够稳定

原始代码直接写死了：

```python
img1, img2, gray1, gray2 = load_images('leuvenA.jpg', 'leuvenB.jpg')
```

但项目目录中并不存在这两张图片，因此实际运行时会失败。

为了适应当前实验图片，改成了默认读取：

1. `2-a.jpg`
2. `2-b.jpg`

同时增加了标准 `main()` 入口，方便独立运行。

### 3. 文件读取失败缺少明确提示

虽然原始代码中有文件不存在的判断，但整体结构偏教学草稿风格，不利于后续整理和复用。因此在新的标注版中保留并明确说明了读取失败检查的重要性。

### 4. 描述子可能为空

如果某张图纹理太少，SIFT 可能返回 `des = None`。此时如果直接进入匹配步骤，会导致程序报错。因此增加了空描述子保护。

### 5. RANSAC 可能估计失败

在匹配数量不足或图像条件较差时，`findFundamentalMat()` 可能返回 `None`。因此需要增加失败保护逻辑。

### 6. 实验结果图需要保存

原始代码只做显示，不做保存。对于课堂展示和实验报告来说，保存结果图是非常有必要的，因此增加了结果图导出逻辑。
