## 一、`feature_matching.py` 应完成的内容

### 代码目标

该脚本应完成一条完整的双图像特征匹配实验流程：

1. 读取两张待匹配图像。
2. 转换为灰度图。
3. 使用 SIFT 提取关键点与描述子。
4. 使用 KNN 进行初始匹配。
5. 使用 Lowe Ratio Test 进行第一轮筛选。
6. 使用 RANSAC 估计基础矩阵并剔除误匹配。
7. 输出匹配数量、内点数量等统计信息。
8. 保存 RANSAC 前后的可视化结果图。

### 实验目标

通过该脚本，应该能够回答以下问题：

1. 两张图像中哪些区域存在稳定可重复的局部特征。
2. 仅靠外观相似得到的匹配中，存在多少误匹配。
3. 引入几何约束后，误匹配能否被有效剔除。
4. 不同图像对之间，为什么匹配效果会不同。

### 本脚本当前应输出的结果

运行后应至少得到：

1. 两张图的关键点数量。
2. Ratio Test 后保留的匹配数量。
3. RANSAC 后保留的内点数量。
4. 基础矩阵 `F`。
5. 两张保存下来的结果图：
   `matches_before_ransac.png`
   `matches_after_ransac.png`

### 修改与扩展方向

后续可以围绕这些方向扩展：

1. 将图片路径改成命令行参数输入。
2. 支持自动缩放大图，加快实验速度。
3. 增加不同匹配策略对比，例如 ORB 与 SIFT 对比。
4. 将实验结果自动写入日志或 Markdown 文件。
5. 增加极线绘制功能，进一步展示基础矩阵意义。

## 二、`virtual_camera_test.py` 应完成的内容

### 代码目标

该脚本应完整演示三维点经过相机模型投影到二维图像的过程：

1. 构造一个简单三维场景。
2. 构造相机内参矩阵 `K`。
3. 设置相机旋转矩阵 `R` 和平移向量 `t`。
4. 将世界坐标变换到相机坐标系。
5. 将相机坐标投影到图像平面。
6. 通过透视除法得到二维像素坐标。
7. 在 2D 和 3D 两个视图中可视化结果。

### 实验目标

通过该脚本，应该能够回答以下问题：

1. 焦距变化会如何影响成像大小。
2. 相机平移会如何影响图像中的目标位置。
3. 相机旋转会如何改变投影视角。
4. 为什么图像坐标系和数学坐标系表现不同。

### 本脚本当前应输出的结果

运行后应至少得到：

1. 相机内参矩阵 `K` 的打印结果。
2. 立方体在二维图像中的投影结果。
3. 立方体与相机位置的三维示意图。

### 修改与扩展方向

后续可以围绕这些方向扩展：

1. 将焦距改成可配置参数。
2. 增加绕 `X`、`Y`、`Z` 轴旋转的实验。
3. 加入多个不同深度的物体，观察透视效应。
4. 将图像投影结果保存到本地。
5. 将当前投影模型与真实相机标定联系起来。
